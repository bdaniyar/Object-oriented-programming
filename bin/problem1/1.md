When to use an interface
Use an interface when:

1. You need a contract (what to do, not how)
	•	defines behavior only

2. You need multiple inheritance
	•	a class can implement many interfaces

3. Classes are not related
	•	but share common behavior

When to use an abstract class
Use an abstract class when:

1. You want shared code + partial implementation
	•	some methods implemented
	•	some abstract

2. There is an “is-a” relationship
	•	subclasses extend a base class

3. You need fields (state)
